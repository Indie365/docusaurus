## Using Markdown

Docusaurus supports Markdown with [Remark](https://github.com/remarkjs/remark) as the Markdown parser and is extensible via plugins.

In the beginning of each of your docs, specify the `id`, the `title`, and `sidebar_label` so that Docusaurus will pick them up correctly when generating your site. We'll talk about that again in a minute.

```markdown
---
id: greeting
title: Hello
sidebar_label: hello
---

# Hello from Docusaurus

Are you ready to create the documentation site for your open source project?
```

## Embedding React Components

Docusaurus has in-house support for [MDX](https://mdxjs.com). This allows you to write JSX within your Markdown files and render them as React components.

Rename your file from `.md` to `.mdx`. Docusaurus will hot reload your dev server and you won't need to restart.

And try this block here:

```jsx
export const Highlighter = ({children}) => (
  <span
    style={{
      background: 'rgb(37, 194, 160)',
      padding: '2px 4px',
      borderRadius: '2px',
      color: 'white',
    }}>
    {children}
  </span>
);

### <Highlighter>Docusaurus</Highlighter> green is my favorite color.
```

Notice how it renders both the markup from your React component and the Markdown syntax:

export const Highlighter = ({children}) => (
  <span
    style={{
      background: 'rgb(37, 194, 160)',
      padding: '2px 4px',
      borderRadius: '2px',
      color: 'white',
    }}>
    {children}
  </span>
);

## Sidebar

To generate a sidebar to your Docusaurus site, you need `sidebars.js` under the root directory. Our template should have already generated it for you, so you may edit it for your own customization.

<!-- TODO: change classic template to use `sidebars.js` from json -->

```bash
.
├── docs
│   └── greeting.md
├── sidebars.js
```

The `sidebars` object requires a `docs` field, under which the keys are the names of each category of your docs, and the values are arrays of `id`'s you specify in your docs' frontmatter.

To add a doc to the sidebar, add the `id` specified in the frontmatter of the doc into its category.

```diff
module.exports = {
  docs: {
+   "Getting started": ["hello"],
    "Docusaurus": ["doc1"],
    "First Category": ["doc2"],
    "Second Category": ["doc3"],
  }
};
```

## Live Editor

You can create live code editors by creating a code block with `live` attached to the language meta string.

    ```jsx live
    function Clock(props) {
      const [date, setDate] = useState(new Date());
      useEffect(() => {
        var timerID = setInterval(() => tick(), 1000);

        return function cleanup() {
          clearInterval(timerID);
        };
      });

      function tick() {
        setDate(new Date());
      }

      return (
        <div>
          <h2>It is {date.toLocaleTimeString()}.</h2>
        </div>
      );
    }
    ```

It will render an interactive editor. Changes to the code will reflect on the result panel live.

